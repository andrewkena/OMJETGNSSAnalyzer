# OMJET GNSS Analyzer — Android (прототип)

Нативное Android-приложение (Kotlin UI) со встроенным Python-интерпретатором
([Chaquopy](https://chaquo.com/chaquopy/)), переиспользующее модуль `core/`
десктопной версии для декодирования `.cnb` и анализа миссии.

## Как открыть и собрать

1. Установите [Android Studio](https://developer.android.com/studio) (включает
   нужный JDK 17, Android SDK/NDK можно поставить через SDK Manager при первом
   запуске).
2. File → Open → выберите папку `android/` (не корень репозитория).
3. Дождитесь Gradle Sync — при первом открытии Android Studio сама создаст
   `gradlew`/`gradle-wrapper.jar`, если их нет. Первая синхронизация скачает
   Chaquopy и prebuilt-колёса `numpy` для Android — нужен интернет.
4. Run ▶ на эмуляторе или подключённом устройстве (minSdk 24 / Android 7.0+).

Если Gradle не находит Python на машине разработчика (нужен для части
Chaquopy-тулчейна, не для самого приложения), укажите интерпретатор явно в
`app/build.gradle.kts`:

```kotlin
chaquopy {
    defaultConfig {
        buildPython("C:/Python312/python.exe") // или путь к своему python3
        ...
    }
}
```

## Как пользоваться

1. «Выбрать папку с миссией...» — выберите папку, где лежат **оба** файла:
   `<имя>.cnb` и `<имя>.cnb.obs` (RINEX OBS, конвертированный заранее внешним
   инструментом — то же требование, что и у десктопной версии).
2. «Анализировать» — запускает Python-конвейер (`mobile_pipeline.py`) в
   фоновом потоке, результат выводится текстом.

## Что уже работает

- Собственный декодер GPS (`RANGECMPB`/`RAWEPHEM`) — тот же код, что и в
  десктопной версии (`core/novatel/`), без изменений.
- Расчёт траектории (`BESTPOS`), длины пути, PDOP по эфемеридам.
- Анализ спутников по группировкам, качество фотосъёмки, итоговая оценка
  миссии — те же критерии, что в десктопном приложении.

## Чего пока нет (известные ограничения MVP)

- **Нет графиков и карты траектории** — `matplotlib` не подключён к мобильной
  сборке (это отдельная задача: либо рендерить графики на стороне Python в
  PNG через Agg-backend, как на десктопе, и показывать как `Bitmap`, либо
  перерисовать графики средствами Android/Compose).
- **Нет генерации PDF** — `reportlab` не включён в `chaquopy { pip { ... } }`.
- **Нет собственного декодирования ГЛОНАСС/Galileo/BeiDou/QZSS** — как и на
  десктопе, для них нужен уже готовый `.obs`/`.nav` от внешнего конвертера.
- Выбор файла — через системный диалог «папка», а не drag-n-drop/история
  последних файлов.

## Структура

```
android/
  app/
    build.gradle.kts          # модуль приложения + конфигурация Chaquopy
    src/main/
      AndroidManifest.xml
      java/com/omjet/gnssanalyzer/
        AnalyzerApp.kt         # инициализация Python при старте приложения
        MainActivity.kt        # UI: выбор папки, запуск анализа, вывод результата
      python/
        mobile_pipeline.py     # точка входа для Kotlin (без matplotlib/reportlab)
        core/                  # копия core/ из десктопной версии (анализ/декодер)
      res/                     # layout, строки, иконки
  build.gradle.kts            # корневой файл проекта
  settings.gradle.kts
```

`android/app/src/main/python/core/` — копия соответствующих файлов из
корневого `core/` (без `pipeline.py`, `*_plot`/`pdf_report`/`report_exporter`,
которые тянут `matplotlib`/`reportlab`). При изменении логики декодирования в
десктопной версии не забудьте скопировать правки и сюда — пока это не
сделано автоматическим shared-источником.
