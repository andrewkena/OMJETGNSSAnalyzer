# OMJET GNSS Analyzer

**Версия:** 0.26  
**Автор:** [andrewkena](https://github.com/andrewkena)

![logo](assets/logo.png)

Программа для анализа «сырых» бинарных данных ГНСС-приёмника (файлы `.cnb`,
формат ComNav/SinoGNSS, OEM-совместимый протокол) с аэрофотосъёмочной миссии
беспилотника. Оценивает качество съёмки с точки зрения последующего
построения ортофотоплана: спутники, точность позиционирования, траекторию,
высоту полёта и интервалы фотосъёмки.

## Возможности

### Декодирование CNB

- Собственный декодер на чистом Python извлекает из бинарного потока:
  - измерения GPS (псевдодальности, фаза, доплер, C/N0 — лог `RANGECMPB`);
  - эфемериды GPS (`RAWEPHEM`, полный битовый разбор подкадров по ICD-GPS-200);
  - готовое позиционное решение приёмника (`BESTPOS`) — для построения траектории;
  - временные метки фотоснимков.
- Поддержка сжатых архивов `.obs.gz` (разворачиваются автоматически).
- Декодер покрывает **только GPS**. ГЛОНАСС/Galileo/BeiDou читаются из
  сконвертированного файла `.obs`/`.nav`.

### Анализ спутников

- Среднее/минимальное/максимальное число видимых спутников по эпохам, в т.ч.
  по каждой группировке (GPS/GLONASS/Galileo/BeiDou/QZSS/SBAS).
- Принимаемые частоты (коды сигналов) по каждой группировке — из заголовка RINEX OBS.
- PDOP по эпохам — рассчитывается из положений GPS-спутников и траектории приёмника.

### Анализ фотосъёмки

- Интервалы между снимками, разрывы, номинальная частота съёмки.
- Сопоставление каждого снимка с числом видимых спутников на момент съёмки и
  присвоение оценки качества (GOOD/NORMAL/LOW).
- Подсветка дублирующихся геометок (совпадающие lat/lon) оранжевым цветом в таблице.

### Интерактивная карта

- Растровая карта с тайлами OpenStreetMap (Tile Server: `tile.openstreetmap.org`),
  загрузка и кэширование тайлов в фоне.
- Точки геометок окрашены по качеству: зелёный / жёлтый / красный.
- Отфильтрованные точки — серые с красным ободом; исключённые вручную — просто серые.
- При наведении на геометку в таблице — точка подсвечивается на карте (и наоборот).
- Подсказка с номером точки при наведении мышью прямо на маркер.

### График высоты полёта

- Встроен снизу карты, высота 60 px.
- Показывает только высоту полёта (без рельефа), метки фотоснимков и
  пунктирную линию средней крейсерской высоты.
- Средняя высота вычисляется только по фазе горизонтального полёта
  (участок выше 60% от диапазона высот между первым и последним подъёмом).
- Масштабирование по оси X — колёсиком мыши относительно положения курсора.
- Ось X подписана в минутах, часовые отметки выделены жирным.

### Итоговая оценка миссии

- Комбинированная оценка (ОТЛИЧНО/ХОРОШО/НОРМАЛЬНО/ПЛОХО) на основе качества
  фотосъёмки, качества GNSS-сигнала и доли хороших геометок.
- В обзоре отображается количество геометок и количество отфильтрованных.
- Средняя высота полёта в разделе «ВЫСОТА ПОЛЁТА» — по крейсерскому участку.

### Звуковое оповещение

- По завершении анализа воспроизводится `assets/done.wav` — как в основном окне,
  так и в режиме Мультитрек.

## Установка

Требуется Python 3.12+.

```bash
git clone https://github.com/andrewkena/OMJETGNSSAnalyzer.git
cd OMJETGNSSAnalyzer
python -m venv .venv
.venv\Scripts\activate      # Windows
pip install -r requirements.txt
```

Либо скачайте готовый `OMJET_GNSS_Analyzer.exe` из раздела Releases — Python и
зависимости не требуются.

---

## Запуск

**Графический интерфейс:**

```bash
python gui.py
```

или запустите `OMJET_GNSS_Analyzer.exe`.

В окне: кнопка «Выбрать файл...» → выберите `.cnb` → «Анализировать».

**Из командной строки** (без GUI, результат печатается в консоль):

```bash
python main.py
```

(путь к `.cnb`-файлу указывается в `main.py` константой `CNB_FILE`).

## Описание интерфейса

| Вкладка | Содержание |
|---|---|
| Обзор миссии | Итоговые оценки, время/длительность полёта, точность позиционирования, высота (крейс.), PDOP |
| Спутники | Статистика по спутникам и группировкам, принимаемые сигналы |
| Качество геометок | Интервалы съёмки, разрывы, итоговое качество, доля хороших геометок |
| Итоговый отчёт | Сводный текстовый отчёт по миссии |
| Графики | Спутники по времени, интервалы, гистограмма, PDOP |
| Файлы результата | Ссылки на CSV/TXT/PDF, кнопки «Просмотр» и «Открыть папку» |

Под вкладками всегда видна панель геометок с таблицей точек и кнопками исключения/восстановления.  
Карта с графиком высоты занимает правую половину окна.

## Фильтрация геометок

Панель фильтров расположена над таблицей геометок. Содержит три независимые строки:

1. **Фильтровать от средней высоты** — исключает геометки, высота которых
   отклоняется от крейсерской средней более чем на заданный процент (вводится вручную).

2. **Фильтровать дубли** — оставляет из каждой группы дублирующихся геометок
   (одинаковые lat/lon) только одну.

3. **Фильтровать по высоте** — исключает геометки вне заданного диапазона
   высот [мин; макс].

Чекбокс **«Применить фильтрацию геометок»**: при экспорте CSV оставляет
в файле только геометки, прошедшие все включённые фильтры.

Отфильтрованные точки отображаются на карте серыми с красным ободом.  
Точки, исключённые вручную через кнопку «Исключить», — просто серыми.

## Мультитрек

Кнопка «Мультитрек» открывает отдельное окно для одновременного отображения
нескольких миссий на общей карте.

- Перетащите файлы `.cnb` в окно или используйте кнопку загрузки.
- Каждый трек отображается своим цветом.
- В правом нижнем углу карты — статистика: число треков, протяжённость,
  площадь покрытия, средняя крейсерская высота.
- Чекбокс «Скрыть статистику» — убирает оверлей без закрытия окна.
- Кнопка «Экспортировать изображение» — сохраняет снимок карты в PNG.
- По окончании загрузки трека воспроизводится `done.wav`.

## Сохраняемые файлы

Сохраняются в подпапках рядом с исходным `.cnb`-файлом:

```
<имя_файла>/
  decoded/   — RINEX OBS/NAV, декодированные данные
  plots/     — графики PNG
  reports/   — photo_satellite_report.csv, mission_report.txt, mission_report.pdf
```

- `photo_satellite_report.csv` — таблица геометок: lat/lon/высота/спутники/качество.  
  При включённом «Применить фильтрацию геометок» — только прошедшие фильтр точки.
- `mission_report.txt` / `mission_report.pdf` — итоговый отчёт с графиками.

## Ограничения

- Расчёт PDOP и собственный декодер OBS/NAV используют только GPS.
- Позиционирование (`BESTPOS`) — решение самого приёмника (обычно одноточечное,
  без RTK/PPK); программа не выполняет собственных вычислений координат.
- Карта требует подключения к интернету для загрузки тайлов; при отсутствии
  соединения отображается сетка без подложки.
- Просмотр PDF открывается во внешней программе по умолчанию.

```
  height_profile_widget.py  # график высоты полёта

assets/                 # логотип, иконка, done.wav
gui.py                  # графический интерфейс (Tkinter)
main.py                 # запуск из командной строки
```
